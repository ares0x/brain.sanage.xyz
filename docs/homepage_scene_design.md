# 首页场景化重构方案

## 一、核心问题

当前首页是「工具列表」思维——用户看到的是一堆分类标签和游戏卡片。

changjing.md 指出的关键洞察：
> 用户每天最容易点开的不是"大脑训练"，而是"救命按钮"。
> 用户一眼就知道"这是现在的我"。

**转变方向**：从「这是什么类型的游戏？」→「我现在需要什么？」

---

## 二、场景划分

基于 changjing.md 的 6 个场景，结合现有 11 个游戏，整合为 **5 个场景**。

（6个太多会导致选择困难，5个是认知心理学中推荐的选项上限附近。）

| # | 场景名（用户语言） | 图标 | 适合时机 | 包含游戏 |
|---|------------------|------|---------|---------|
| 1 | **我想进入状态** | 🌅 | 开工前、学习前、下午犯困 | 舒尔特方格、反应速度、抗干扰训练、凝视启动 |
| 2 | **我要专心工作** | 🎯 | 需要长时间专注、屏蔽干扰 | 专注力追踪、冲动控制、抗干扰训练 |
| 3 | **我脑子很乱** | 😵 | 刷完手机后、信息过载、走神了 | 专注拉回、4-7-8呼吸法、凝视启动 |
| 4 | **我想训练大脑** | 🧠 | 有成长目标、主动提升 | 工作记忆、数字广度、专注切换 |
| 5 | **我想放松** | 🌙 | 睡前、疲惫、焦虑 | 4-7-8呼吸法 |

### 为什么这样划分？

**1. 我想进入状态**（changjing.md「开始工作前」）
- 这是最强的日常入口，用户每天开工前都会想到
- 关键词：启动、进入状态、点火
- 包含热身类轻量游戏

**2. 我要专心工作**（changjing.md「进入深度专注」）
- 知识工作者最痛的点：干扰太多、无法进入心流
- 与「进入状态」的区别：前者是热身，后者是持续专注

**3. 我脑子很乱**（changjing.md「大脑疲劳/注意力崩塌时」+「快速恢复专注」）
- changjing.md 说这是"极高频需求"，"救命按钮"
- 合并"注意力崩塌"和"快速恢复"，因为它们本质都是"从混乱回到专注"
- 这个场景视觉权重应该更高（或用不同配色强调）

**4. 我想训练大脑**（changjing.md「主动训练大脑」）
- 传统认知训练，面向有明确成长目标的用户
- 强调：长期训练、认知成长、高难度

**5. 我想放松**（changjing.md「睡前/放松恢复」）
- 用户不一定只想"变强"，还想"恢复"
- 目前只有 4-7-8 呼吸法，后续可扩展冥想、身体扫描等

### 关于游戏跨场景

同一个游戏可以出现在多个场景：
- **抗干扰训练(Flanker)** → 场景1（热身）+ 场景2（专注）
- **凝视启动** → 场景1（开工前启动）+ 场景3（混乱后重启）
- **4-7-8呼吸法** → 场景3（快速平复）+ 场景5（睡前放松）

这是合理的，因为用户在不同状态下确实会用同一个工具达到不同目的。

## 四、场景区块设计

每个场景是一个可展开的卡片区块：

```
┌─────────────────────────────────────────┐
│ 🌅  我想进入状态                    4项  │  ← 点击展开/折叠
│     开工前、学习前，5分钟热身              │
├─────────────────────────────────────────┤
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ │
│  │ 舒尔特   │ │ 反应速度 │ │ 抗干扰   │ │
│  │ 方格     │ │ 测试     │ │ 训练     │ │
│  │          │ │          │ │          │ │
│  │ 开工前热 │ │ 了解你的 │ │ 帮你在嘈 │ │
│  │ 身...    │ │ 大脑...  │ │ 杂环境.. │ │
│  └──────────┘ └──────────┘ └──────────┘ │
│  ┌──────────┐                            │
│  │ 凝视启动 │                            │
│  └──────────┘                            │
└─────────────────────────────────────────┘
```

### 设计细节

1. **默认展开状态**：
   - 场景1「我想进入状态」默认展开（因为这是最强日常入口）
   - 其余场景默认折叠
   - 用户点击场景头部展开/折叠

2. **场景头部**（可点击区域）：
   - 左侧：emoji 图标 + 场景名（用户语言，粗体）
   - 下方：一行小字说明适合时机
   - 右侧：游戏数量 + 展开/折叠箭头
   - 背景：白色卡片，带左边框色条（每个场景不同颜色）

3. **场景左边框色条**：
   - 🌅 我想进入状态 → 暖黄色 `#D4A832`
   - 🎯 我要专心工作 → 蓝色 `#5A9DE0`
   - 😵 我脑子很乱 → 珊瑚色 `#D4847C`（强调色，因为这个场景最高频）
   - 🧠 我想训练大脑 → 紫色 `#A87AD4`
   - 🌙 我想放松 → 青绿色 `#6BA7A8`

4. **游戏卡片在场景区块内**：
   - 使用现有的游戏卡片样式
   - 网格布局：移动端 2 列，桌面端 3 列
   - 保留 benefit 副标、时长、难度

## 五、关键设计决策

### 决策1：为什么不用「救命按钮」单独设计？

changjing.md 建议做一个巨大的「重新集中」按钮。

**我的判断**：先不做单独的大按钮，而是把「我脑子很乱」场景放在视觉更突出的位置（比如不同的边框色、或在 Today Plan 下方第一个位置）。原因：
1. 保持一致性，5个场景统一格式更简洁
2. 「我脑子很乱」场景内有 3 个选项（Stroop、呼吸法、凝视启动），比单一大按钮更灵活
3. 如果数据证明用户高频点击这个场景，可以再升级为一个固定入口

### 决策2：为什么游戏可以跨场景？

抗干扰训练(Flanker)既在「进入状态」又在「专心工作」。

**理由**：
- changjing.md 明确说 Flanker 既是开工前热身游戏，也是深度专注训练
- 用户在不同状态下对同一个游戏的心理预期不同
- 跨场景不会导致重复感，因为每个场景的上下文不同

### 决策3：为什么场景1默认展开？

「我想进入状态」是 changjing.md 认证的"非常强的日常入口"。

默认展开降低了一击成本，用户打开首页就能看到具体游戏。

## 八、风险评估

| 风险 | 缓解措施 |
|------|---------|
| 用户习惯了旧分类，找不到游戏 | 游戏仍在场景中展示，且更贴近用户心理状态 |
| 游戏跨场景导致重复感 | 跨场景的游戏只有 3 个（Flanker、凝视启动、呼吸法），重复度可控 |
| 场景展开/折叠增加交互成本 | 场景1默认展开，其余场景头部高度只有 60px，不占用太多空间 |
| 5个场景太多，选择困难 | 场景使用用户语言，用户能一眼识别"这是现在的我" |
